# Security Policy

This document outlines the security policy and reporting procedures for **Kavun**.

## Reporting a Vulnerability

The **Kavun** team takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings and will make every effort to acknowledge your contributions promptly.

If you discover a security vulnerability within this project, please **do not** open a public issue. Instead, report it through the secure channel specified below:

* **GitHub:** Please utilize the [GitHub Security Advisories](https://github.com/) tab on the repository to report vulnerabilities privately.
* **Alternative:** If private advisories are unavailable, contact the maintainer directly via the communication channels listed in the Code of Conduct.

After your report is received, the maintainers will investigate the issue and keep you informed of the progress toward a fix and full public announcement.

### Third-Party Dependencies

Please report security bugs in third-party dependencies directly to the respective maintainers of those modules.
